# Spike Preservation, Context Requirements, and Variable Artifact Structure

## Motivation

For the planned deep-learning-based gradient artifact correction, a central methodological risk is that the model may learn shortcuts that are not physiologically meaningful. This is particularly relevant when synthetic training pairs are generated by combining clean spike EEG with extracted fMRI artifact windows.

The primary concern is not only whether the model removes gradient artifacts, but whether it preserves epileptiform activity correctly and avoids introducing spike-like structures into spike-free data.

## Core Problem

A naive supervised setup would train the model on pairs of short windows:

- `noisy = clean + artifact`
- `target = clean` or `target = artifact`

While this is technically straightforward, it creates several scientific risks if the window generation process is not designed carefully.

### Risk 1: Positional Bias of Spikes

If spike events are always placed at a fixed location within the training window, the model may implicitly learn that spike-like activity is expected at this position. In that case, successful spike preservation would no longer be a genuine signal-preservation capability, but partly a consequence of positional prior information that is unintentionally encoded in the dataset.

This creates two problems:

1. The model may fail when spikes occur at different temporal positions within the artifact-contaminated interval.
2. The model may hallucinate spike-like structures because it has learned that a spike is statistically likely at a specific position.

For the intended application, this is unacceptable. The model should preserve a spike only if a spike is actually present in the observed data.

### Risk 2: Overfitting to Fixed Artifact Window Length

The extracted artifact windows from one fMRI dataset have a specific window length defined by the trigger structure and correction pipeline. However, this length is not guaranteed to be constant across datasets, acquisition protocols, or scanners.

If the model is trained only on one fixed artifact duration, it may overfit to this temporal structure. Such a model may then generalize poorly when applied to data with different trigger spacing, different artifact durations, or modified acquisition parameters.

### Risk 3: Lack of Context

A single isolated short window may not contain enough information to reliably distinguish between:

- a true epileptic spike,
- a residual artifact fluctuation,
- a sharp but physiologically irrelevant transient,
- or noise introduced during synthetic data generation.

Gradient artifacts are not random isolated events. They are typically coupled to the MRI sequence and therefore exhibit temporal regularity. Spikes, in contrast, are transient events that may occur at arbitrary times relative to the artifact cycle.

This means that the distinction between neural activity and artifact cannot always be solved robustly from a strictly local single-window view alone.

## Consequences for Dataset Design

The synthetic dataset should therefore not be constructed as a collection of fixed, perfectly aligned, center-anchored spike windows.

Instead, the following design principles are required.

### 1. Spike Position Must Vary Within the Window

If spike annotations mark event onsets, the spike should not always appear at the same offset relative to the synthetic artifact window. During dataset generation, the spike onset should be placed at varying temporal positions within the allowed window.

This reduces the risk that the model learns a trivial positional prior instead of real spike preservation.

### 2. Spike-Free Windows Must Be Included

If the model is trained mostly or exclusively on windows containing spikes, it may learn that spike-like morphology is generally expected. To counter this, the training dataset must also contain clean windows without spikes.

This is essential to evaluate and train against false positive spike introduction.

### 3. Artifact Structure Should Be Augmented

The artifact should not be treated as a perfectly fixed waveform. Even if extracted from real data, the artifact windows should be varied during training, for example through:

- amplitude scaling,
- temporal jitter,
- random cropping or shifting,
- and, if feasible, varying effective artifact duration.

This makes the training distribution more robust and reduces overfitting to one highly specific acquisition setup.

Temporal jitter must be implemented carefully. Circular shifting with wrap-around can create non-physiological boundary patterns, because the end of an artifact epoch is copied to the beginning. A safer choice is non-circular shifting with edge filling or cropping.

### 4. Variable-Length Inference Must Be Anticipated

In deployment, the model should not assume that all future datasets contain artifacts of exactly the same length as the training examples. Therefore, the system should be designed around chunk-wise processing of variable-length inputs rather than hard-coding one exact temporal artifact template.

### 5. Dataset Offsets Should Not Become Learning Targets

Different EEG datasets can contain different baseline offsets due to subject-specific, amplifier-specific, or preprocessing-specific factors. These offsets are not the artifact morphology that the model should learn. If the model is allowed to learn dataset-specific DC shifts, it can subtract an artificial baseline component during correction.

Therefore, training and inference should apply consistent local mean removal:

- demean each input epoch per channel,
- demean the artifact target during training,
- and remove the mean of the predicted artifact before subtraction.

This makes the task focus on artifact shape and timing rather than absolute voltage offset.

## Consequences for Model Design

The model should not be designed as if it receives a single isolated event and must reconstruct a deterministic spike location. Instead, it should learn to separate artifact and neural signal based on temporal structure and contextual evidence.

### Preferred Output Formulation

A strong formulation is to predict the artifact rather than the cleaned EEG directly:

- input: `noisy`
- output: `artifact_hat`
- reconstruction: `clean_hat = noisy - artifact_hat`

This is advantageous because it reduces the tendency to generate plausible-looking clean EEG that was not actually supported by the input.

### Preferred Architecture Characteristics

The following properties are desirable:

- fully convolutional temporal models,
- support for variable input lengths,
- channel-wise operation as a robust baseline,
- optional context window larger than the final prediction region.

A particularly sensible strategy is to use a longer input context and predict only the center region. This allows the model to exploit surrounding temporal structure without forcing the target event to appear at a fixed position.

## Need for Additional Context

A key conceptual insight is that a single short window may be insufficient to fully disambiguate chance fluctuations from structured artifact.

Therefore, the model may benefit from additional information beyond the immediate target segment.

Possible context sources include:

- larger temporal input windows,
- neighboring consecutive epochs,
- relative position to the MRI trigger,
- slice or volume phase information,
- or explicit trigger-derived conditioning variables.

The rationale is that gradient artifacts are sequence-coupled and thus partially predictable from temporal context, whereas spikes are not strictly phase-locked to the MRI cycle.

## Implications for Evaluation

A model cannot be considered successful if it merely lowers signal energy or suppresses visible artifact amplitude. It must also be evaluated with respect to spike preservation and spike hallucination.

Relevant evaluation dimensions include:

- preservation of true spikes,
- suppression of false artifact transients,
- avoidance of spurious spike generation,
- robustness to varying spike positions,
- and robustness to varying artifact lengths.

This means that artifact-removal metrics alone are insufficient. The evaluation protocol must explicitly include spike-sensitive performance criteria.

## Summary

The central challenge is not only to remove gradient artifacts, but to do so without encoding artificial positional priors about epileptiform events.

For this reason, the training dataset should:

- contain paired noisy and target windows,
- include both spike-containing and spike-free examples,
- vary spike position within the window,
- augment artifact morphology and timing,
- and prepare the model for variable-length inference.

Correspondingly, the model should be designed to use temporal context and should preferably predict artifact components rather than directly hallucinating a cleaned EEG trace.

## Thesis-Ready Core Statement

A naive synthetic training setup, in which epileptic spikes always occur at a fixed temporal position within an artifact window, risks introducing strong positional priors into the model. Such priors can impair generalization and may lead to undesired hallucination of spike-like activity. To avoid this, the training dataset must vary spike position within the window, include spike-free control examples, and expose the model to temporally augmented artifact patterns. In addition, because fMRI artifact duration is not constant across datasets, the inference design should support variable-length temporal input rather than relying on a fixed artifact window size.

Additional experimental note: the first implementation showed that augmentation and preprocessing choices can themselves introduce artifacts. Circular jitter and zero-padded convolutions produced boundary-dominated predictions, while missing mean normalization allowed the model to learn dataset-specific offsets. These findings support local mean-zero normalization and edge-safe augmentation as methodological requirements, not merely implementation details.
